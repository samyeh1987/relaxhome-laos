## 1. 产品概述

### 1.1 产品定位

一款面向 C 端消费者的**上门专业按摩服务 O2O 平台**，连接有专业资质的按摩技师与有家庭/酒店上门服务需求的用户，提供预约、派单、服务、结算的全流程数字化管理。

### 1.2 产品愿景

> 让每个人在家就能享受到专业、安全、透明的按摩服务体验。

### 1.3 核心价值主张

| 对用户 | 对技师 | 对平台 |
|--------|--------|--------|
| 无需出门，随时预约 | 灵活接单，自主排班 | 双边网络效应 |
| 实名认证，服务透明 | 收益保障，结算及时 | 数据驱动精细化运营 |
| 评价可追溯，放心消费 | 专业背书，职业提升 | 品牌溢价 + 抽佣收益 |

## 2. 市场分析

### 2.1 行业背景

- 中国健康养生行业规模 2025 年预计突破 **8 万亿元**
- 上门 O2O 服务渗透率持续提升，疫情后居家养生习惯固化
- 东郊到家、泰式 APP、按摩到家等平台验证市场需求
- 一线城市高压白领、中年群体、孕产妇是核心需求人群

### 2.2 竞品对比

| 维度 | 东郊到家 | 泰式 APP | 本产品规划方向 |
|------|----------|----------|---------------|
| 服务品类 | 中式推拿为主 | 泰式按摩 | 多品类（中式/泰式/精油/足疗） |
| 技师认证 | 基础资质核验 | 一般 | 严格三级认证 + 定期复审 |
| 安全机制 | 一般 | 一般 | 行程共享 + 紧急 SOS |
| 预约方式 | 即时 + 预约 | 即时为主 | 即时 / 计划 / 套餐订阅 |
| 用户层次 | 大众 | 大众 | 大众 + 精英会员分层 |

### 2.3 目标市场

- **Phase 1（0-12 月）**：聚焦 1-2 个一线城市（北京/上海）冷启动
- **Phase 2（13-24 月）**：扩展至 5-8 个新一线城市
- **Phase 3（25+ 月）**：全国化布局，并探索东南亚出海

## 3. 用户角色与画像

### 3.1 用户端（消费者）

**主力人群 A — 都市白领**
- 年龄：25-40 岁，月收入 1 万以上
- 痛点：久坐肩颈痛、没时间出门、对陌生环境有顾虑
- 需求：即时预约、信任保障、多人拼单

**主力人群 B — 中老年家庭用户**
- 年龄：45-65 岁，由子女代付
- 痛点：身体慢性疼痛，出行不便
- 需求：操作简单、家庭账号共享

**细分人群 C — 孕产妇**
- 专属服务需求（孕期/产后按摩）
- 对技师资质要求极高

### 3.2 技师端（服务方）

**类型 A — 兼职技师**
- 有执照但无固定坐班，希望灵活增收
- 诉求：接单自由度高、结算及时

**类型 B — 全职平台技师**
- 主要收入来源为平台订单
- 诉求：稳定接单量、平台流量扶持

### 3.3 管理员（运营端）

- 平台运营人员、客服、合规审核、财务结算

## 4. 核心功能模块

### 4.1 用户端 APP

#### 4.1.1 首页与服务发现
- 服务品类卡片（中式推拿 / 泰式按摩 / 精油 SPA / 足疗 / 孕产专区）
- 附近技师实时地图（LBS 展示距离、评分、空闲状态）
- Banner 营销位 + 今日推荐技师

#### 4.1.2 预约下单
- **即时单**：30 分钟内可上门（系统自动派单）
- **预约单**：指定日期时间段（提前最长 7 天）
- 服务时长选择（60/90/120 分钟）
- 上门地址管理（家庭地址 / 酒店 / 自定义）
- 备注需求（力度、重点部位）

#### 4.1.3 技师选择
- 技师主页（照片、资质证书、服务次数、评分、评价）
- 技师预约日历（可用时间段）
- 收藏技师 / 指定技师下单

#### 4.1.4 订单管理
- 订单状态实时追踪（待接单 → 已接单 → 技师出发 → 服务中 → 已完成）
- 技师实时位置地图
- 一键联系技师（加密电话 / 平台内消息）
- 紧急 SOS 按钮（一键报警 + 行程共享）

#### 4.1.5 支付与钱包
- 微信支付 / 支付宝 / 银行卡
- 平台钱包余额（充值享折扣）
- 电子发票申请

#### 4.1.6 评价系统
- 服务完成后 24 小时内可评价
- 星级评分 + 标签评价 + 文字/图片
- 技师回评功能

#### 4.1.7 会员体系
- 普通会员 / 银卡 / 金卡 / 钻石卡
- 积分兑换、优先派单、专属折扣
- 套餐包（如：10 次卡、月度无限次）

### 4.2 技师端 APP

#### 4.2.1 认证入驻
- 手机号实名注册
- 上传资质证书（按摩师证、健康证）
- 人脸活体认证 + 后台人工审核

#### 4.2.2 接单管理
- 抢单大厅（附近新订单通知）
- 我的工单（待服务 / 服务中 / 历史）
- 排班设置（开放接单时间段）
- 一键开始/结束服务（自动计时）

#### 4.2.3 导航服务
- 内嵌地图导航（高德 / 百度地图）
- 上门打卡（LBS 验证到达位置）

#### 4.2.4 收益中心
- 每日/每月收益统计
- 提现申请（T+1 到账）
- 账单明细（订单收入 - 平台佣金）
- 绩效激励（完成量排名奖励）

#### 4.2.5 个人主页管理
- 编辑自我介绍、服务项目、照片
- 查看用户评价
- 查看服务统计数据

### 4.3 管理后台（Web 端）

#### 4.3.1 技师管理
- 入驻审核（资质材料核验）
- 技师评分监控 + 违规处理
- 封号 / 降级 / 奖励

#### 4.3.2 订单管理
- 全平台订单实时监控
- 异常订单介入处理
- 退款申请审核

#### 4.3.3 财务管理
- 资金流水总览
- 佣金结算记录
- 提现审批管理
- 对账报表导出

#### 4.3.4 内容运营
- Banner / 弹窗配置
- 优惠券 / 活动管理
- 用户消息群发

#### 4.3.5 数据报表
- DAU / MAU 用户活跃度
- 订单完成率、客单价、复购率
- 技师服务质量排行
- 城市/区域热力分析

#### 4.3.6 风控系统
- 异常行为检测（恶意刷单、虚假评价）
- 用户投诉处理
- 合规内容审核（技师图片、评价内容）

## 5. 业务流程设计

### 5.1 核心下单流程

```
用户选择服务品类
    ↓
填写上门地址 + 预约时间
    ↓
选择技师（指定/系统派单）
    ↓
确认订单信息
    ↓
在线支付（资金托管至平台）
    ↓
系统推送至附近符合条件技师
    ↓
技师接单确认（超时自动推下一位）
    ↓
用户收到接单通知 + 技师信息
    ↓
技师出发 → 用户可实时追踪位置
    ↓
技师到达打卡 → 开始服务计时
    ↓
服务完成 → 技师点击结束
    ↓
用户确认完成 → 触发结算
    ↓
双方相互评价
    ↓
技师收益 T+1 到账
```

### 5.2 技师入驻审核流程

```
技师提交申请资料
    ↓
系统自动校验（身份证 OCR、证书真伪）
    ↓
人工复审（1-3 个工作日）
    ↓
通过 → 激活账户，可接单
失败 → 通知原因，可重新提交
```

### 5.3 退款仲裁流程

```
用户申请退款
    ↓
填写退款原因（服务未开始/质量问题/技师迟到等）
    ↓
系统判断（服务未开始 → 自动全退）
    ↓
需人工介入 → 客服介入调查（查看行程记录/录音备案）
    ↓
判定结果 → 执行退款或驳回
    ↓
双方收到处理结果通知
```

## 6. 技师管理体系

### 6.1 三级认证标准

| 级别 | 要求 | 权益 |
|------|------|------|
| 普通技师 | 按摩师证 + 健康证 + 实名认证 | 接单资格，基础流量 |
| 优选技师 | 普通 + 评分 ≥ 4.8 + 完成单量 ≥ 50 | 首页推荐，优先派单 |
| 明星技师 | 优选 + 评分 ≥ 4.9 + 完成单量 ≥ 200 | 专属主页，高端客户匹配 |

### 6.2 技师绩效激励

- **月单量达标奖**：完成 50/100/200 单分别奖励 ¥200/500/1000
- **好评率奖**：月好评率 ≥ 95% 额外奖励 ¥100
- **新城市拓荒奖**：新城市开荒技师额外补贴
- **违规惩罚**：差评率 > 10% 降级，投诉超 3 次封号

### 6.3 技师行为规范

- 服务过程中禁止录音录像（平台统一备案除外）
- 着装规范，携带平台工具包
- 禁止私下收费或引导线下交易
- 禁止向用户索要个人联系方式

## 7. 安全合规设计

### 7.1 用户安全

- **行程共享**：服务开始后自动向紧急联系人发送位置链接
- **紧急 SOS**：用户一键触发，平台客服 + 用户紧急联系人同时收到警报
- **加密通信**：用户与技师的通话通过平台号码中转，双方真实号码不泄露
- **服务录音备案**（可选）：用户开启后，录音加密存储 30 天，仅纠纷时调取

### 7.2 技师安全

- 平台为技师提供意外险保障（服务期间）
- 异常订单标记（深夜/偏远地址）提示技师谨慎接单

### 7.3 合规要求

- 服务内容严格限定专业按摩（系统关键词过滤，人工抽查评价内容）
- 技师性别与用户性别匹配可选（用户可设置偏好）
- 涉及孕产妇/儿童服务需特殊资质认证
- 遵守各地对养生行业的地方性法规

### 7.4 数据安全

- 用户地址数据加密存储
- 遵守《个人信息保护法》，最小化数据收集原则
- 定期安全审计 + 渗透测试

## 8. 盈利模式

### 8.1 主要收入来源

| 收入类型 | 说明 | 预估占比 |
|----------|------|----------|
| 服务抽佣 | 每单按服务费 15-25% 抽取平台佣金 | 65% |
| 会员订阅 | 月卡/年卡会员费（¥39/月，¥299/年） | 15% |
| 套餐销售 | 预付次数卡/体验包 | 10% |
| 广告投放 | 技师主页竞价排名，品牌合作 | 5% |
| 增值服务 | 技师培训课程、保险产品 | 5% |

### 8.2 财务模型（初步预测）

**假设：城市上线 6 个月后稳定运营**
- 日均订单：500 单
- 客单价：¥200
- 平台抽佣率：20%
- 月营收 = 500 × 200 × 20% × 30 = **¥60 万/月/城市**

## 9. 技术架构选型

### 9.1 客户端

| 端 | 技术选型 |
|----|----------|
| 用户端 APP | React Native（iOS + Android 双端）|
| 技师端 APP | React Native（复用用户端组件库）|
| 管理后台 | React + Ant Design Pro |
| 小程序 | 微信小程序（轻量版，用于裂变传播）|

### 9.2 服务端

| 模块 | 技术选型 |
|------|----------|
| 主业务服务 | Node.js（NestJS 框架）|
| 订单调度引擎 | Go（高并发处理）|
| 实时通讯 | WebSocket（Socket.io）|
| 数据库 | PostgreSQL（主库）+ Redis（缓存）|
| 地理位置服务 | 高德 LBS API + PostGIS |
| 消息推送 | 极光推送 / 阿里云短信 |
| 支付 | 微信支付 + 支付宝开放平台 |
| 对象存储 | 腾讯云 COS |
| 身份核验 | 腾讯云实名认证 API |
| 部署 | 腾讯云 TKE（Kubernetes）|

### 9.3 安全基础设施

- HTTPS 全链路加密
- JWT + OAuth 2.0 认证
- 接口防重放攻击（签名机制）
- 敏感操作二次验证（短信 OTP）
- WAF + DDoS 防护

## 11. 风险评估

### 11.1 合规风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 服务内容越界 | 高 | 严格内容审核 + 关键词过滤 + 人工巡查 |
| 无照经营 | 中 | 要求技师持证上岗，平台核验存档 |
| 数据隐私合规 | 中 | 对标 PIPL，隐私协议更新，最小化数据收集 |

### 11.2 运营风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 技师供给不足 | 高 | 冷启动期补贴技师，与职业院校合作 |
| 用户信任建立慢 | 高 | 强化安全功能，前期主打口碑营销 |
| 恶意刷单/套现 | 中 | 风控模型 + 人工审核 |
| 技师私下揽客 | 中 | 规范合同 + 检测机制 + 惩罚机制 |

### 11.3 竞争风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 东郊到家等头部竞品压制 | 高 | 差异化定位（安全/品质），避免正面价格战 |
| 技师被挖角 | 中 | 提升技师收益，增加平台黏性（成长体系）|

## 附录

### A. 关键术语

- **即时单**：用户下单后 30 分钟内匹配到技师并上门
- **预约单**：用户提前选定日期时间，系统提前匹配派单
- **资金托管**：用户支付后资金暂存平台，服务完成后结算至技师
- **T+1 结算**：技师收益于服务完成次日结算到账
